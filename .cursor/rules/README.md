# Cursor Rules

Optimized `.mdc` rules for Swift/SwiftUI development.

## Structure

```
.cursor/rules/
├── style/
│   ├── formatting.mdc    # Swift standards, SwiftUI usage, code organization
│   └── naming.mdc        # Naming conventions
├── architecture/
│   └── patterns.mdc      # MVVM, design patterns, project structure
├── swift/
│   └── best-practices.mdc # Types, concurrency, property wrappers
├── data/
│   └── state.mdc         # Data storage, state management
├── ui/
│   └── development.mdc   # Apple HIG, accessibility, layout
├── error-handling/
│   └── exceptions.mdc    # Error handling patterns
├── performance/
│   └── optimization.mdc  # Thread, memory, network performance
├── security/
│   └── guidelines.mdc    # Data protection, network security
├── testing/
│   ├── unit-tests.mdc    # Unit test structure
│   └── integration-tests.mdc # Integration testing
├── documentation/
│   ├── comments.mdc      # Code comments
│   └── api-docs.mdc      # API documentation
├── features/
│   └── essential.mdc     # Deep linking, notifications, localization
├── app-store/
│   └── guidelines.mdc    # App Store submission
├── workflow/
│   └── development.mdc   # Development workflow
└── system/
    └── validation.mdc    # Rule debugging
```

## Best Practices Applied

- **No Redundancy**: Each concept appears in exactly one file
- **Simple and Focused**: Short, actionable rules
- **Organized**: Logical grouping by category
- **Generic**: No project-specific references
